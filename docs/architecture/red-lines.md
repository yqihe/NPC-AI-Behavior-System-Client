# 禁止事项（红线）

## 禁止客户端做 AI 决策

- **禁止**在客户端代码中实现 NPC 行为逻辑（状态转换、威胁评估、行为树执行）。所有 AI 决策由服务端完成，客户端只负责表现
- **禁止**客户端预测或插值 NPC 的 FSM 状态变化。以 world_snapshot 为唯一权威数据源
- **禁止**客户端缓存 NPC 状态并在断线时使用缓存数据驱动行为。断线 = NPC 行为冻结

## 禁止硬编码游戏数据

- **禁止**在 C# 代码中硬编码 NPC 类型名称（如 `if (type == "civilian")`）。NPC 类型可能随时通过运营平台增加
- **禁止**在 C# 代码中硬编码事件类型名称。使用配置或服务端下发的数据
- **禁止**在 C# 代码中硬编码 FSM 状态名称做逻辑分支。视觉表现（动画/特效）的状态映射应通过配置表驱动

## 禁止协议违规

- **禁止**修改消息信封格式（`{type, id, data}`）。协议由服务端定义，客户端遵循
- **禁止**发送服务端不认识的消息类型。只使用 `spawn_npc`、`remove_npc`、`publish_event`、`query_npc`
- **禁止**忽略 `error` 类型响应。必须展示给用户（Toast/日志），不能静默吞掉

## 禁止过度设计

- **禁止**实现客户端侧的 NPC 寻路/碰撞检测。服务端当前不处理移动，位置是静态的
- **禁止**实现复杂的网络层（重连队列、消息确认、序号排序）。WS 是可靠传输，简单重连即可
- **禁止**在毕设阶段实现多语言/本地化
